# Agent Evolution v2 — Hermes 自我进化系统

## 设计理念

**v1 的问题：**
- 分析 nanobot 的历史日志 → 目标错位
- 用正则匹配"错误""失败"关键词 → 假阳性
- 建议空泛不可执行 → 永远 0 个改进被应用
- cron 只跑 `--analyze` → 闭环断裂

**v2 的变化：**
- 分析 Hermes **自身**的数据 → 自指分析
- 结构化解析 cron 输出、技能库状态、git 日志 → 精确分析
- 生成具体可执行的改进方案 → 能自动执行的就自动执行
- 分级执行：低风险自动、中风险出方案、高风险标记讨论

## 架构

```
┌─────────────────────────────────────┐
│          数据源（Hermes 自身）        │
│  • Cron 任务输出 (~/.hermes/cron/)  │
│  • 技能库 (~/.hermes/skills/)       │
│  • Git 历史 (git log)               │
│  • 文件变更                          │
└──────────────┬──────────────────────┘
               │
    ┌──────────▼──────────┐
    │   CronHealthAnalyzer │ ← Cron 任务健康分析
    │  SkillsQualityAudit  │ ← 技能库质量审计
    │   GitChangeAnalyzer  │ ← Git 变更模式分析
    │    ActionPlanner     │ ← 生成可执行改进方案
    │      AutoFixer       │ ← 自动执行低风险改进
    │  ReportGenerator     │ ← 生成人类可读报告
    └──────────┬───────────┘
               │
    ┌──────────▼──────────┐
    │  P0 → 自动执行       │
    │  P1 → 生成方案       │
    │  P2 → 标记讨论       │
    └─────────────────────┘
```

## 文件结构

```
v2/
├── hermes_self_analyzer.py  # 主分析器（含所有模块）
└── output/
    ├── evolution-report-*.md  # 每日进化报告
    ├── evolution-plan-*.json  # 改进计划
    └── auto-fix-*.json        # 自动执行记录
```

## 使用方法

```bash
# 手动运行
cd ~/projects/agent-evolution
./scripts/evolve.sh --v2

# 或直接运行
python3 v2/hermes_self_analyzer.py
```

## 自动执行的改进类型

| 类型 | 风险级别 | 示例 |
|------|---------|------|
| 补全缺失 SKILL.md | 低 | 创建基础文档模板 |
| 修复格式问题 | 低 | 清理空标签、统一格式 |
| 脚本改进 | 中 | 生成具体 diff，需审核 |
| 架构变更 | 高 | 标记问题，等待讨论 |

## 与 v1 对比

| 维度 | v1 | v2 |
|------|-----|-----|
| 数据源 | nanobot 历史 | Hermes 自身 |
| 分析方式 | 正则关键词匹配 | 结构化数据解析 |
| 改进建议 | 空泛文字描述 | 具体文件路径+步骤 |
| 执行能力 | 仅交互确认 | 分级自动执行 |
| 验证方式 | 重扫描同文件 | 对比 cron 前后指标 |
| 闭环状态 | ❌ 只分析不执行 | ✅ 分析+自动执行 |
