采用四层分层架构：

1. **数据采集层**: 执行记录、错误日志、资源监控

## 版本说明

### 版本号格式

遵循 [语义化版本 2.0.0](https://semver.org/)：

```
主版本号。次版本号。修订号
MAJOR.MINOR.PATCH
```

- **MAJOR**: 破坏性变更
- **MINOR**: 向后兼容的功能新增
- **PATCH**: 向后兼容的 Bug 修复

### 发布周期

- **主版本**: 每 3-6 个月
- **次版本**: 每月
- **修订版**: 按需

### 发布流程

1. 更新版本号 (`src/__version__.py`)
2. 更新本变更日志
3. 创建 Git 标签
4. 发布 GitHub Release
5. 发布到 PyPI (如果适用)
